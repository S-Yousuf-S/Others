## 📌 Project Overview
 
**Sensei** is a next-generation, AI-powered learning companion designed to replace passive reading with interactive, contextual exploration. Built entirely in Python using Streamlit and the Google Gemini API, it operates as a lightweight **Retrieval-Augmented Generation (RAG)** system.
 
Users can upload their study materials (Presentations, Code Notebooks, and PDF notes) and interrogate them via a conversational interface. Sensei provides a dual-layer response: a concise, AI-generated explanation, backed immediately by the raw, retrieved text from the user's own documents.
 
The primary engineering goal of Phase 1 was to build a robust, transparent retrieval engine **without** relying on heavy Vector Databases, focusing instead on logic, string manipulation, chunking algorithms, and UI explainability.

## 🚀 Core Features & Document Handling
 
### 📄 Multi-Format Document Ingestion
Sensei currently supports three highly distinct file formats, each requiring custom parsing logic:
- **PDFs (`.pdf`):** Parsed using `PyPDF2`, with pages joined by double newlines so each page becomes a natural chunk boundary for retrieval. Aggressive text normalisation repairs broken sentences. All extracted text is passed through a Unicode surrogate sanitizer to prevent rendering crashes.
- **Jupyter Notebooks (`.ipynb`):** JSON-level extraction that cleanly separates Markdown text from executable Python code, rendering code in visually distinct fenced Python blocks. Surrogate characters are sanitized here too.
- **PowerPoint (`.pptx`):** Iterates through slide shapes to extract text. Features **Slide-Aware Parsing**, ensuring each slide acts as a self-contained atomic chunk (labelled `[Slide N]`) so content from different slides never bleeds together during retrieval.
### 🧠 The Dual-Layer Response Engine
Instead of just giving an answer, Sensei proves its work:
1. **The Tutor Summary:** A concise, easy-to-understand explanation generated by Gemini based strictly on your documents.
2. **The Source Expanders:** Expandable UI elements colour-coded by document type (🟥 PDF, 🟩 Notebook, 🟦 Slides) that display the exact chunks of text or code the AI used to formulate its answer.
### 🔌 Smart API Error Handling
The API connection panel provides specific, actionable feedback rather than a generic failure message:
 
| Error Code | Message Shown | Cause |
|---|---|---|
| `429` | Quota Exceeded | Free tier rate limit hit — wait and retry |
| `500` | Internal Error | Gemini service temporarily unavailable |
| `400` | Invalid Model | Model string deprecated or unavailable |
| Other | Connection Failed | Invalid API key or network issue |
 
### 🥷 Dojo Mode UI
A custom-built, premium CSS framework that overrides standard Streamlit styling. It introduces:
- A deep forest-green background with teal and gold radial glows at each corner.
- Alternating teal (`#3DD9B4`) and gold (`#D4AF37`) heading colours with matching text-shadow glows.
- An animated gold-to-teal gradient strip pinned to the top of the viewport.
- Custom scrollbars, glowing input boxes, gold-gradient buttons, and styled chat message bubbles.
- A carefully chosen dark-theme colour palette designed to reduce eye strain during late-night study sessions.

## 🛠️ Architecture & Developer Notes
 
| Concern | Implementation |
|---|---|
| **Data Storage** | No Vector DBs. Uploaded files are text-extracted and serialised into `sessions/documents_{session_id}.json` files to keep the root directory clean. |
| **Retrieval Engine** | Keyword-based matching with stop-word filtering. Uses a sliding **Context Window** algorithm to grab adjacent text chunks for PDF and notebook content. Slides bypass this entirely — each `[Slide N]` chunk is returned as-is to prevent cross-slide contamination. |
| **PDF Chunking** | Pages are joined with `\n\n` rather than `""`, so each page becomes a natural chunk boundary. This significantly improves retrieval precision on multi-page documents. |
| **Unicode Safety** | A `sanitize_text()` function strips illegal Unicode surrogate characters (`U+D800–U+DFFF`) from all extracted text before storage. These appear in some PDFs and notebooks and would otherwise crash Streamlit's markdown renderer. |
| **Notebook Safety** | Custom regex routines strip Base64-encoded images from Notebook markdown before rendering, replacing them with a `*[Image omitted from text]*` placeholder to prevent massive data blobs cluttering the UI. |
| **Session Isolation & Cleanup** | Implements `uuid` tagging to ensure multi-user instances do not overwrite each other's data. A time-based garbage collector automatically deletes abandoned session files older than 2 hours on the next startup. Legacy root-level `chat.json` and `documents.json` files from earlier versions are also automatically migrated and removed on first run. |
| **Storage Guards** | A 10MB soft limit is enforced on the combined document payload before writing. Chat history is capped at the last 50 turns to prevent unbounded file growth. |
| **Prompt Engineering** | System prompts strictly confine the LLM to answer *only* using the provided context, heavily reducing AI hallucination. |

## 💻 Installation & Setup
 
### Prerequisites
 
Ensure you have **Python 3.10+** installed.
 
**1. Clone the repository and navigate to the directory:**
```bash
git clone https://github.com/S-Yousuf-S/Sensei-AI.git
cd Sensei-AI
```
 
**2. Create and activate a virtual environment (Recommended):**
```bash
python -m venv sensei_env
# Windows:
sensei_env\Scripts\activate
# macOS / Linux:
source sensei_env/bin/activate
```
 
**3. Install dependencies:**
```bash
pip install -r requirements.txt
```
 
**4. Launch the application:**
```bash
streamlit run sensei_fs_v9.0.py
```
 
> 💡 **Pro-Tip for Windows / Virtual Environment Users:**
> If your system throws a "command not found" error, or if you want to guarantee you are using the exact dependencies installed in your active virtual environment, use this command instead:
> ```bash
> python -m streamlit run sensei_fs_v9.0.py
> ```
 
*(Once running, paste your Gemini API key into the sidebar to connect the engine.)*

## 🙋 Frequently Asked Questions
 
**Q: Why doesn't Sensei use a Vector Database (like Pinecone or ChromaDB)?**<br>
**A:** This is a deliberate architectural choice for Phase 1. By relying on lightweight JSON serialisation and keyword-chunking, Sensei remains highly portable, requires zero cloud infrastructure, and serves as an excellent foundational study in how Retrieval-Augmented Generation (RAG) logic actually works under the hood.
 
**Q: I got a `ResourceExhausted 429 Quota Exceeded` error. What happened?**<br>
**A:** You have hit the Gemini API rate limit on the free tier. Sensei will display a specific "Quota Exceeded" message in the sidebar rather than a generic error. Take a short break — your quota resets on a per-minute and per-day basis depending on Google's current limits.
 
**Q: The Connect API button shows an "Invalid Model" error. What does that mean?**<br>
**A:** This means the `gemini-3-flash-preview` model string returned a `400` error from Google's API. This can happen if the model is temporarily unavailable or has been deprecated. Check [Google AI Studio](https://aistudio.google.com) for current model availability.
 
**Q: Is my API key safe?**<br>
**A:** Yes. Your API key is injected directly into the Streamlit session state and passed securely to Google's servers. It is never logged, saved to disk, or visible anywhere in the codebase.
 
**Q: Can other users on the app see my uploaded study materials?**<br>
**A:** No. Sensei v9.0 uses UUID-namespaced sessions. Your uploaded document text and chat history are isolated to your specific browser session. Furthermore, session files older than 2 hours are automatically deleted from the server to ensure total data privacy.
 
**Q: Is there a file size limit for uploads?**<br>
**A:** Yes. Sensei has a built-in soft guard of `10MB` for the combined extracted text payload per session — this limit is also shown directly on the upload widget. If your upload exceeds it, the app displays a friendly warning in the sidebar rather than crashing silently.
 
**Q: Why do my PDF Source Expanders have so much text in them?**<br>
**A:** Sensei is designed to over-retrieve rather than under-retrieve for PDF and notebook content. If it finds a keyword match, it grabs the surrounding paragraphs (the Context Window) to ensure Gemini has all the nuance needed to give a complete answer. Slides are the exception — only the exact matching slide is returned, with no context window expansion.
 
**Q: Why don't images from my PowerPoint slides show up in the expander?**<br>
**A:** For Phase 1, Sensei utilises a lightweight text-extraction pipeline. While it perfectly reads and chunks all the text on your slides, images embedded in PowerPoint slides are not described or extracted. Multimodal image analysis is planned for future updates.
 
**Q: My PDF content looks garbled or has strange symbols in it. What happened?**<br>
**A:** Some PDFs — particularly those exported from older software or generated programmatically — contain illegal Unicode surrogate characters (`U+D800–U+DFFF`) that corrupt text rendering. Sensei v9.0 includes a `sanitize_text()` routine that automatically strips these from all extracted content before it is displayed or stored.
 
**Q: I asked a question but got "❌ Not found in uploaded material". Why?**<br>
**A:** Sensei's retrieval engine uses keyword matching — it searches for the exact words from your question inside your documents. If your question uses different terminology than what appears in the notes (synonyms, abbreviations, or paraphrasing), no chunks will score a match. Try rephrasing using the exact words that appear in your uploaded material.
 
**Q: Can I upload multiple files at the same time?**<br>
**A:** Yes. You can upload multiple files in a single session. Files of the same type are merged — two PDFs become one searchable corpus, two notebooks are combined, and multiple presentations are pooled. All files count together toward the 10MB session limit.
 
**Q: Does Sensei remember my conversation if I refresh the page?**<br>
**A:** Yes, within the same browser session. Your chat history and uploaded documents are saved to session-specific JSON files on the server. Refreshing the page without closing the tab fully restores your conversation. However, closing the browser and returning later starts a new session, since the session ID lives in the browser tab's memory.
 
**Q: What happens if I upload a scanned PDF?**<br>
**A:** Scanned PDFs are image-based and contain no extractable text layer. `PyPDF2` will extract empty strings from these pages, meaning Sensei will have no content to search. For best results, use text-based PDFs — ones where you can select and copy text directly in any PDF viewer.
 
**Q: Will you add support for Word or Excel files?**<br>
**A:** Yes! Excel (`.xlsx`) and Word (`.docx`) integration is planned for the next major development phase.
